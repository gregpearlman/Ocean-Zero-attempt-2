# VC HQ — template skeleton

Copy the **contents** of this folder into a fresh private repo to start your own VC HQ:

```bash
cp -r starter-kit/templates/. /path/to/your-new-repo/
```

This is a working static site you can open directly (`open index.html`) — it renders the
example data in `data.js`. No build step, no `npm install`.

## What's here

| Path | What it is |
|---|---|
| `index.html` | Page skeleton: sidebar nav + one section per view |
| `styles.css` | Design system (tokens in `:root` — change these to re-skin) |
| `app.js` | Minimal working renderer + hash router + edits.js overlay merge |
| `data.js` | **Edit this** — `COMPANIES` / `INVESTORS` / `PAIRINGS` with example records |
| `edits.js` | Empty in-browser-edit overlay (written by the save-edits function) |
| `netlify.toml` | Deploy config + security headers |
| `scripts/build-meta.sh` | Netlify build entry point (version.js + history) |
| `scripts/sync-live-data.sh` | Restores `edits.js` from origin/main if the pre-push hook blocks you |
| `netlify/functions/save-edits.js` | Serverless backend for in-browser editing (set `GITHUB_REPO`) |
| `.githooks/pre-push` | Guardrail: blocks any push that would revert `edits.js` |
| `.claude/settings.json` | Pre-allows safe commands; activates the pre-push hook each session |
| `.claude/skills/` | Two example reusable Skills (governance-tab, pitchbook-investor-profile) |
| `CLAUDE.md` | **Fill this in** — the handover brief Claude reads every session |

## First steps after copying

1. Replace the example records in `data.js` with your portfolio + first investors.
2. Fill in every `<PLACEHOLDER>` in `CLAUDE.md`.
3. Set `window.SITE_VERSION` will be generated by Netlify; ignore locally.
4. `node --check data.js && node --check app.js` to confirm syntax.
5. Commit, push, connect Netlify (see `../docs/04-netlify-deploy.md`).

## Growing it

The skeleton renders Dashboard / Companies / Investors / Pairings. To add a view:
add a `<section class="view" id="view-x">` + a nav link in `index.html`, a render
function in `app.js`, and register it in the `VIEWS` map. To add a data file
(e.g. an outreach tracker), drop a `<script>` tag **before** `app.js` in `index.html`
and read its global in your renderer.
