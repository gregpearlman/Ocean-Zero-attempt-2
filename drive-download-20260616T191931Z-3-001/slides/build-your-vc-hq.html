The product

What you're building

A password-protected internal site that is the single source of truth for your fund's live deal book — and a tool Claude keeps current from your inbox, Slack and Drive.

Dashboard

Round-status tiles per portfolio company, sorted hot→cold. Click to expand.

Fund Performance

Capital deployed, NAV, MOIC and per-company returns from your master sheet.

To-Do

Operational + outreach streams, per team member, refreshed weekly.

Investors & Pairings

Your investor universe × companies, ranked and tier-scored with citable sources.

Companies

Each portco as its own mini-site: round, cap table, board, governance, competitors, exit comps.

Outreach trackers

Live investor-outreach sheets synced from Drive, searchable + filterable.

Design philosophy

Three principles that make this work

01

Plain static site

Just HTML, CSS, JS. No framework, no build, no npm. Data lives in plain .js files. Anyone — and any Claude session — can edit it instantly.

02

A living tool

Not a one-off report. Most sessions mine fresh context from Gmail / Slack / Drive and update the data files, then push.

03

Claude-operated

A CLAUDE.md handover makes any fresh Claude session productive in minutes. Routines keep it updated unattended.

The whole point: push to GitHub → Netlify auto-deploys in ~30–60s. Editing the fund's brain is as cheap as a git commit.

Before you start

What you'll need

Accounts

  • GitHub — a private repo for your fund
  • Netlify — free tier deploys; Pro ($19/mo) adds visitor-password protection
  • Claude — Claude Code (CLI / web / desktop) for editing + routines

Connectors (optional but recommended)

  • Gmail, Slack, Google Drive MCP connectors
  • Fireflies (meeting transcripts) if you use it

Decisions to make up front

  • Your list of portfolio companies + a stable id for each
  • Your team and their initials (owner chips)
  • Which data sources are in-scope to mine (and which are strictly off-limits)
No coding background required. You drive Claude in plain English; the templates in this kit give it the scaffold.
1

Create the GitHub repo

Make it private — this repo holds confidential deal data.

# Use the templates/ folder from this kit as your starting point
gh repo create your-fund/vc-hq --private
cd vc-hq
cp -r /path/to/starter-kit/templates/* .
cp -r /path/to/starter-kit/templates/.claude .claude
git add -A && git commit -m "scaffold vc-hq"
git push -u origin main
  • The templates/ folder ships a working skeleton: index.html, styles.css, app.js, data.js, plus deploy + guardrail config.
  • Keep the repo private forever. Treat every file as confidential.

Step 3 · the core data model

Three arrays drive everything

COMPANIES

Your portfolio. Each has an id, sector, stage, round state, milestones.

INVESTORS

Your funding universe — funds + angels with research, ticket size, stage/geo tags.

PAIRINGS

Every investor × company match, ranked by tier + conviction, with a killer angle and intro path.

The pairing schema (abridged)

{ rank:1, tier:1, investor:'lowercarbon', company:'acme-fusion',
  conviction:5, contact:'Clay Dumas, Partner',
  killerAngle:'Backed 3 fusion plays 2024–25; deep-tech lead.',
  checkAsk:'$15–20M lead', introPath:'Warm via …',
  ticketMatch:'yes', stageMatch:'yes',
  verificationUrl:'https://…' }   // citable 2024–26 source
Discipline that pays off: every pairing carries a citable verificationUrl. It keeps the universe honest and stops fabricated funds creeping in.
4

Deploy to Netlify + lock it down

Connect & deploy

  • In Netlify, "Add new site → Import from GitHub" and pick your repo.
  • Build command + publish dir come from netlify.toml (already in the template).
  • Every git push to main now auto-deploys.

Password-protect it

  • Netlify Pro: Site config → Visitor access → set a site password.
  • Share the password out-of-band with your team only.
  • The template sets X-Robots-Tag: noindex so it never gets crawled.
# netlify.toml (shipped in the template)
[build]
  publish = "."
  command = "bash scripts/build-meta.sh"
[functions]
  directory = "netlify/functions"
5

Onboard Claude with CLAUDE.md

This is the single most important file. Claude reads it automatically at the start of every session — it's your fund's operating manual.

  • Start from templates/CLAUDE.md and fill in the <PLACEHOLDERS>: fund name, site URL, repo, portfolio table, team matrix, guardrails.
  • Keep it current. When the architecture changes, update the handover in the same commit.
  • It contains: project overview, file map, deploy steps, per-company state, sensitivities, and your common workflows.
A good CLAUDE.md means a brand-new Claude session is productive in <5 minutes — no re-explaining your fund every time.
6

Connect your data sources

Wire up MCP connectors so Claude can read your real context — the difference between a static brochure and a living tool.

ConnectorWhat Claude does with it
GmailMine investor / portco threads for fresh state, last-touch dates, follow-ups
SlackRead deal channels; drive auto-update routines

Set boundaries explicitly in CLAUDE.md: an allow-list of what's in-scope to mine and a default-deny for everything else (other funds, personal mail).
7

The mining workflow

How a typical "update the site" session goes — and how to make it fast.

  1. 1. Search Gmail + Slack for a company: from:partner@fund.vc Acme
  2. 2. Synthesize new state into the per-company narrative doc
  3. 3. Update data.js (round size, lead, status) + the dashboard status object in app.js
  4. 4. git commit && git push → auto-deploys
Parallelize: for a full sweep, ask Claude to spawn one background agent per company. 9 portcos mined at once instead of in series.
8

Live data integrations

Connect Drive spreadsheets so the site mirrors your real working trackers — no manual re-entry.

Outreach trackers

  • Each company can have an investor-outreach sheet on Drive.
  • A small script parses it → <company>-outreach.js → searchable table on that company's page.
  • A diff-gated refresh commits only when the data actually changed.

Fund master sheet

  • Your cap-table / cash-flow sheet → fund-performance.js.
  • Drives the Fund Performance view: paid-in, NAV, MOIC, projections.
  • Refreshed by a weekly routine; committed so it survives deploys.
9

Automate with routines

Claude routines (claude.ai/code/routines) run on a schedule and keep the site fresh while you sleep.

Slack scanner

Watches a #website channel; treats each message as an edit instruction → commits + pushes → replies with the SHA.

Email scout

Sweeps the inbox (within your allow-list) and posts proposed updates to Slack for a human ✅ before applying.

Drive sync

Weekly pull of each outreach tracker + the fund sheet; diff-gated commit.

Dedup & verify passes

Periodic agent sweeps to re-verify investor partners + citations.

Each routine has a cursor file so it never re-processes old messages, and commit footers carry the source link for dedup.
10

In-browser edit mode

Let non-technical teammates fix data right on the live site — no git, no Claude.

  • A Netlify Function (save-edits) writes an overlay file (edits.js) straight to the repo via the GitHub API.
  • Protected by a shared EDIT_KEY + a fine-grained GitHub token, both stored only in Netlify env vars.
  • The app layers edits.js over data.js at runtime, so edits show instantly.
Guardrail: a committed pre-push hook stops any code push from ever reverting edits.js — it's the only store of those edits. Always rebase before pushing.
Non-negotiables

Guardrails & sensitivities

Security

  • Never commit .netlify/ or any token — .gitignore covers it.
  • Secrets (GitHub PAT, edit key, service keys) live only in Netlify env vars.
  • Keep investor emails out of any externally-shareable layer.

Data integrity

  • Encode sensitive deal context as explicit "never externalize" rules in CLAUDE.md.
  • Every new pairing needs a citable source — no fabricated funds or LinkedIn slugs.
  • Flag, don't silently "fix", anything that looks factually wrong.
Write your guardrails down as a numbered list in CLAUDE.md. Claude follows them on every session; they compound in value.
Power-ups

Reusable Claude Skills

Point it at an SHA / financing doc → it extracts share classes, bodies and decision rules into the Governance tab.

The kit ships these two as working examples. Build your own for any task you repeat.
